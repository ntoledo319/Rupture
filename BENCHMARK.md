## Schema

Each row records: `repo`, `commit`, `kit`, `result` (`pass | fail | warn`), `findings`, `duration_ms`. The full JSON is at `docs/status/benchmark.json`.

## Methodology

- Read-only clone of each repo at the pinned commit.
- Each kit runs in `--strict` mode against the matched file globs.
- A `pass` means the kit ran clean and produced no severity ≥ 7 findings.
- A `warn` means findings exist but none are severity 10.
- A `fail` means a severity-10 finding (e.g. a deadline already in the past).
- No PRs are opened from the benchmark run. PRs only happen on repos that have explicitly installed the Rupture GitHub App (see `apps/github-app/`).

## Corpus

The corpus is curated in `corpus/public-repos.txt`. To propose an addition or removal, open an issue tagged `corpus`. Repos that include a `.no-rupture` file at root are excluded.
