## Before you redeploy (the API now fails closed)

Production is **already live**, so `/home/ubuntu/sites/eolkits-api/.env.production`
on the VPS already contains the live secrets — they are **reused** on redeploy,
not re-entered:

```
STRIPE_KEY, STRIPE_WEBHOOK_SECRET, GITHUB_APP_ID, GITHUB_APP_PRIVATE_KEY,
GITHUB_WEBHOOK_SECRET, RESEND_API_KEY, PUBLIC_SITE_URL, PUBLIC_API_URL, EOLKITS_API_PORT
```

The hardening added **two new vars** that must be appended to that same file or
startup will abort in production (it now validates the full set):

```
# Set once and keep stable; signs short-lived internal upload refs (SSRF mitigation).
EOLKITS_INTERNAL_URL_SECRET=$(openssl rand -hex 32)

# The App's URL handle; powers the /pack/install button. Confirm the real slug at
# github.com/settings/apps (repo docs say "eolkits-migration-bot" but that public
# page 404s, so verify). Non-blocking: /pack/install falls back without it.
GITHUB_APP_SLUG=eolkits-migration-bot
```

Note: these secrets are NOT reachable from the dev environment (no SSH to the
VPS; GitHub Actions secret values are write-only; the Stripe MCP can act with the
live key but never exposes its raw value). They live only on the VPS and in your
Stripe/GitHub/Resend dashboards.
